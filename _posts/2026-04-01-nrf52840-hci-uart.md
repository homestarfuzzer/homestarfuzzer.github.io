Before you can do anything interesting with BLE — sniff traffic, enumerate GATT services, intercept writes — you need a working adapter. Most guides skip this part. They assume you already have one.

This post covers getting there: flashing Zephyr HCI UART firmware onto a Nordic nRF52840 dongle (PCA10059) and verifying BLE scanning works on Kali Linux. GATT enumeration and active target interaction come in the next post.

---

## Why the nRF52840

The common USB dongles people reach for — CSR8510, whatever shipped in an Amazon 3-pack — hit range and capability limits fast. The nRF52840 is the serious option: Coded PHY (extended range), advertising extensions, passive sniffing. But it ships without HCI firmware. It shows up as a dead USB device out of the box. You have to build and flash it yourself.

That's what this covers.

> **HCI primer:** HCI (Host Controller Interface) is the protocol layer between your OS and a Bluetooth controller. `hci_uart` is HCI over UART — what we're flashing here. At the end you want a working `hci0` interface showing `UP RUNNING`.

---

## What you need

- Kali Linux (tested on 2025.x, Python 3.13)
- Nordic nRF52840 dongle (PCA10059)
- ~8 GB free disk space

---

## Step 1 — Install nrfutil

Nordic's CLI for packaging and flashing firmware over USB DFU.

```bash
wget https://developer.nordicsemi.com/.pc-tools/nrfutil/x64-linux/nrfutil
chmod +x nrfutil
mv nrfutil /usr/local/bin/nrfutil
nrfutil install nrf5sdk-tools
nrfutil install device
```

---

## Step 2 — Install build dependencies

```bash
apt install -y cmake ninja-build bluetooth bluez bluez-tools
pip install west --break-system-packages
```

`west` is Zephyr's meta-tool — workspace init, source pulls, builds.

---

## Step 3 — Set up the Zephyr workspace

```bash
mkdir ~/ble-dongle && cd ~/ble-dongle
west init zephyrproject
cd zephyrproject
west update
```

`west update` pulls the full Zephyr source tree. Budget 10–15 minutes and don't interrupt it.

```bash
pip install -r zephyr/scripts/requirements.txt --break-system-packages --ignore-installed
west zephyr-export
```

> **Disk warning:** Zephyr is large. If you're on a VM with a tight partition, run `watch df -h /` in a second terminal while this pulls.

---

## Step 4 — Install Zephyr SDK 1.0.0 (ARM only)

Zephyr 4.4.0-rc1 requires SDK 1.0.0 minimum. Older versions fail at the CMake step.

```bash
cd ~
wget https://github.com/zephyrproject-rtos/sdk-ng/releases/download/v1.0.0/zephyr-sdk-1.0.0_linux-x86_64_minimal.tar.xz
tar xf zephyr-sdk-1.0.0_linux-x86_64_minimal.tar.xz
rm zephyr-sdk-1.0.0_linux-x86_64_minimal.tar.xz
cd zephyr-sdk-1.0.0
./setup.sh
```

When `setup.sh` prompts you:

| Prompt | Answer |
|---|---|
| Install GNU toolchain? | `y` |
| Install for all targets? | `n` |
| arm-zephyr-eabi? | `y` |
| All other toolchains? | `n` |
| Install LLVM toolchain? | `n` |
| Install host tools? | `y` |
| Register Zephyr SDK CMake package? | `y` |
| Create symbolic links? | `y` |

ARM only. Everything else is dead weight.

## Step 5 — Build HCI UART firmware

```bash
cd ~/ble-dongle/zephyrproject
west build -b nrf52840dongle/nrf52840 zephyr/samples/bluetooth/hci_uart
```

Takes 2–5 minutes. A successful build ends with:

```
[253/253] Linking C executable zephyr/zephyr.elf
```

Output hex:

```
~/ble-dongle/zephyrproject/build/zephyr/zephyr.hex
```

---

## Step 6 — Package the firmware

DFU flashing requires a `.zip` package, not a raw hex.

```bash
cd ~/ble-dongle

nrfutil nrf5sdk-tools pkg generate \
  --hw-version 52 \
  --sd-req=0x00 \
  --application ~/ble-dongle/zephyrproject/build/zephyr/zephyr.hex \
  --application-version 1 \
  firmware.zip
```

Expected output: `Zip created at firmware.zip`

---

## Step 7 — Flash the dongle

Plug in the dongle. LED should pulse red — that's DFU bootloader mode. If it isn't, press the reset button once.

Confirm it's visible:

```bash
lsusb | grep Nordic
# Expected: 1915:521f Nordic Semiconductor ASA Open DFU Bootloader

ls /dev/ttyACM*
# Expected: /dev/ttyACM0
```

Flash it:

```bash
nrfutil nrf5sdk-tools dfu usb-serial -pkg firmware.zip -p /dev/ttyACM0
```

Success: `Device programmed.`

Confirm the USB PID changed:

```bash
lsusb | grep Nordic
# Expected: 2fe3:0004 NordicSemiconductor Zephyr HCI UART sample
```

If you still see `1915:521f`, it didn't take. Press reset and retry.

---

## Step 8 — Bring up the HCI adapter

```bash
hciattach /dev/ttyACM0 any 115200 flow
rfkill unblock all
hciconfig hci0 up
hciconfig -a
```

You want `UP RUNNING` in the output. RF-kill blocking is common in VMware VMs — `rfkill unblock all` handles it.

---

## Step 9 — Verify BLE scanning

```bash
hcitool lescan
```

Nearby BLE devices should appear within a few seconds. `Ctrl+C` to stop.

If you're seeing devices, you're done.

---

## Every boot — re-attach sequence

`hciattach` doesn't persist across reboots. Run this after every plug-in:

```bash
hciattach /dev/ttyACM0 any 115200 flow
rfkill unblock all
hciconfig hci0 up
```

Worth aliasing.

---

## Known quirks

**BD Address showing `00:00:00:00:00:00`** — normal for HCI UART firmware. Adapter works fine.

**"Can't read local name" error** — cosmetic. Ignore it.

**RF-kill block** — especially common in VMware. Always run `rfkill unblock all` after attaching.

**CMake fails during build** — almost always an SDK version mismatch. Confirm you're on SDK 1.0.0.
