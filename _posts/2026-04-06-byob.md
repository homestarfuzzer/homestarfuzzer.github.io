Working in security long enough, you start to notice the pattern. Someone gets excited about hacking. They watch a few videos, take a course, then try to actually practice — and immediately hit a wall. Not a firewall. Just setup.

Which version of Docker? Is WSL2 configured right? Where do I get DVWA again? Why is Juice Shop throwing a port conflict? Half an hour later, the momentum's gone.

That's the problem BYOB solves.

## What it does

BYOB gives you a browser dashboard at `localhost:1337` where every vulnerable lab is a button. Click start, wait for the container to pull, you're in. Click stop, it disappears. Click NUKE when you're done — images, containers, gone.

No config file. No CLI flags. No accounts anywhere.

```bash
git clone https://github.com/homestarfuzzer/BYOB.git
cd BYOB
npm install
npm start
```

Open `http://localhost:1337`. That's it.

Windows users: run inside WSL2, or double-click `scripts/start.bat`.

## What's included

The goal was to cover the full learning arc, not just one category. BYOB ships with twelve labs across four areas:

**Web application hacking**

| Lab | What you practice |
|---|---|
| OWASP Juice Shop | OWASP Top 10, SQLi, XSS, API abuse |
| DVWA | SQLi, XSS, File Inclusion, Command Injection |
| WebGoat | Guided lessons, XXE, Deserialization, JWT |
| bWAPP | 100+ vulns including SSRF, IDOR, Clickjacking |
| OWASP NodeGoat | Node.js stack, Prototype Pollution |
| Damn Vulnerable GraphQL | GraphQL introspection, injection, batching attacks |
| Altoro Mutual | Realistic banking app sim, SQLi, XSS |

**API security**

| Lab | What you practice |
|---|---|
| crAPI | OWASP API Top 10, BOLA, BFLA |
| VAmPI | REST API exploitation, Mass Assignment |
| Pixi | API auth bypass, BOLA |

**Network / system**

| Lab | What you practice |
|---|---|
| Metasploitable 2 | Multi-service exploitation, Metasploit |
| Attack Box | Pre-built attack toolkit with browser terminal |

**CTF infrastructure**

| Lab | What you practice |
|---|---|
| CTFd | Host your own CTF or practice with challenges |

Running everything at once is about 5–6 GB. Images download once and cache locally. NUKE wipes it all instantly.

## The Metasploitable 2 + Attack Box combo

This pairing deserves its own section because it's the closest thing to a full network pentest scenario you can run locally.

Metasploitable 2 exposes ~20 exploitable services — FTP, SSH, Telnet, HTTP, MySQL, and more. Attack Box is a lightweight Kali container (~900 MB) that comes preloaded with `nmap`, `metasploit`, `sqlmap`, `hydra`, `nikto`, and `gobuster`, plus a browser-based terminal so you never leave the dashboard.

Start Metasploitable 2 — BYOB shows you the target IP — click **Launch Attack Box** — terminal opens in your browser — type `nmap <target>` and go.

```bash
# Inside the Attack Box browser terminal
nmap -sV 172.20.0.2
msfconsole
use exploit/unix/ftp/vsftpd_234_backdoor
set RHOSTS 172.20.0.2
run
```

Both containers run on an isolated Docker bridge network (`cyberlab-net`, `172.20.0.0/24`). Nothing touches your real LAN.

## Adding a lab

You don't touch any code. Edit `labs/labs.json` and add an entry:

```json
{
  "id": "my-lab",
  "name": "My Vulnerable App",
  "category": "Web",
  "difficulty": "Beginner",
  "description": "What this lab teaches.",
  "image": "docker/image:tag",
  "imageSize": "~200 MB",
  "networkMode": "bridge",
  "ports": [{ "host": 9000, "container": 9000 }],
  "url": "http://localhost:9000",
  "tags": ["XSS", "SQLi"],
  "resources": [
    { "label": "Docs", "url": "https://example.com" }
  ]
}
```

Restart BYOB and your lab shows up in the dashboard. That's the full contribution path — no Node.js knowledge required.

## Setup

Prerequisites: Node.js 18 LTS or higher, Docker Desktop (latest).

```bash
git clone https://github.com/homestarfuzzer/BYOB.git
cd BYOB
npm install
npm start
```

If Docker isn't installed, BYOB detects it and shows the right install steps for your OS. On Windows, run inside WSL2 or use the included `.bat` launcher.
